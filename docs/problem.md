# EasyView Morph Engine — Prompt-Driven Website Personalization

## 1. Business Scenario
You are an AI Engineer at EasyView, an accessibility-focused browser extension whose mission is "Clarity for Every Brain."
Today, every website on the internet follows a one-size-fits-all approach. Whether users browse YouTube, Gmail, LinkedIn, Reddit, government portals, banking websites, research papers, or educational platforms, the experience is decided entirely by the platform.
The problem is that users are not the same.
A student preparing for exams, a professional working under time pressure, a dyslexic reader, a user with ADHD, a senior citizen, and a researcher all consume information differently, yet they are forced to use the same interface.
For years, developers and platforms controlled how users experienced the web. Users could only make limited adjustments through browser settings, accessibility toggles, or custom extensions.
With modern AI systems capable of understanding natural language instructions, users should be able to personalize any website without writing code.
EasyView wants to build a system that allows users to describe how they want a website to behave, and have the interface adapt automatically.

---

## 2. Problem Statement
Build EasyView Morph Engine.
A prompt-driven website adaptation system that allows users to customize and transform any website using natural language instructions.

Example prompts:
• Make this page easier to understand
• Convert this page into study mode
• Hide YouTube Shorts
• Make this website senior friendly
• Reduce cognitive overload
• Convert this website into a Minecraft theme
• Make this page distraction free

The system should:
• Interpret user intent from natural language prompts
• Generate transformation instructions
• Modify webpage structure, layout, styling, and content presentation
• Apply accessibility-focused adaptations when required
• Preserve website functionality while improving usability and personalization
